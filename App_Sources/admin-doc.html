Démarrage

Au premier lancement, sélectionnez le dossier racine du projet (celui qui contient App_Sources et Campagnes).

L'application utilise l'API File System. Chrome ou Edge sont requis ; Firefox n'est pas supporté.

Pour changer de dossier : cliquez sur le chemin affiché en bas de page puis « MODIFIER ».

Pilotage

Vue Kanban en 3 colonnes : En cours d'évaluation, À bilanter, Clôturés.

Filtrez par superviseur ou cherchez un agent par nom/site. Le pourcentage d'avancement s'affiche en haut à droite.

Depuis « En cours » : cliquez + Évaluer pour saisir les notes. Depuis « À bilanter » : Traiter le bilan pour rédiger et envoyer le bilan qualité.

Un agent « Clôturé » peut être remis à bilanter via le lien dédié sur sa carte.

Saisie – Évaluations

Renseignez Agent, Campagne, Offre, Date. La grille de critères s'affiche : attribuez une note (0–10) et un commentaire par critère. Survolez les libellés pour voir les barèmes.

Le commentaire global peut être généré par IA (bouton dédié) ou saisi manuellement.

Enregistrez pour créer l'évaluation. Plusieurs évaluations par agent sont gérées via les onglets (+ pour en ajouter).

Saisie – Bilan

Onglet Bilan : synthèse des évaluations. Un brouillon peut être enregistré pour reprendre plus tard.

Le bilan est généré par IA à partir des évaluations. Vous pouvez le modifier avant envoi.

Une fois envoyé, l'agent passe en colonne « Clôturés ». Remettre à bilanter le replace dans « À bilanter ».

Dashboard

Statistiques par campagne : KPIs, graphiques, répartition par site et offre.

Utilisez les filtres Site et Offre pour affiner les données.

Admin – Campagnes

Créez des dossiers de campagne (ex. 2026 S1, 2026 S2) qui apparaîtront dans la sidebar Pilotage et Dashboard.

Chaque dossier contient les évaluations et bilans des agents pour la période.

Admin – Grille

La Grille est accessible depuis le hub admin via Référentiels et paramètres.

Modifiez les catégories et critères de la grille d'évaluation. Glissez-déposez pour réordonner.

Pour chaque critère : libellé, barème (min/max), tooltip affiché au survol lors de la saisie.

Pensez à forcer la sauvegarde après vos modifications.

Admin – Référentiels et paramètres

Depuis le hub admin, la tuile Référentiels et paramètres ouvre une page avec six modules : Agents, Superviseurs, Sites, Durée des communications, Types d'offres, Grille.

Agents, Superviseurs et Sites : gestion des listes de référence (profils, débriefeurs, sites). Durée des communications et Types d'offres : paramètres utilisés dans les formulaires. La Grille : catégories, critères, barèmes et tooltips de la grille d'évaluation.

Chaque module ouvre une page dédiée (sections détaillées ci-dessous).

Admin – Durée des communications

Définissez les seuils de durée (court / moyen) pour les communications. En dessous du premier seuil = badge vert (courte), entre les deux seuils = badge orange (moyenne), au-dessus du second = badge rouge (longue).

Ces seuils sont utilisés pour afficher les indicateurs de durée dans les vues Pilotage et Saisie.

Admin – Types d'offres

Liste des offres (Employeurs, TI-AE, CESU, PAJEMPLOI, etc.) utilisées dans les formulaires d'évaluation. Ajoutez ou supprimez des entrées selon vos besoins, puis sauvegardez.

Admin – Prompt IA

Deux templates de prompt : commentaire d'évaluation et bilan qualité. Placeholders : {{agentName}}, {{campaignName}}, {{evalsCount}}, {{criteriaBlock}}, {{evaluationsBlock}}, etc.

Modifiez le ton et le format des textes générés selon vos consignes.

Admin – Paramètres système

Cette page regroupe la clé API Mistral (optionnelle, pour les boutons « Générer avec l'IA » du bilan et du commentaire d'évaluation) avec l'option pour désactiver l'IA, et la source des mises à jour (dépôt GitHub). Configurez propriétaire du dépôt, nom du dépôt, et éventuellement un token (lecture seule) pour un dépôt privé. La vérification et l'installation des mises à jour se font depuis le hub admin (bandeau si une version plus récente est disponible).

Lors de l'installation, le dossier Campagnes et les fichiers de données (agents, sites, superviseurs, config, grille) ne sont jamais modifiés.

Après une mise à jour, rechargez la page pour utiliser la nouvelle version.
